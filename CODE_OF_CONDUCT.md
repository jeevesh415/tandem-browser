# Code of Conduct

## Our Standard

Tandem Browser is a technical project, but collaboration still needs basic
professional standards. Participants are expected to be respectful, direct, and
constructive in issues, pull requests, discussions, and review feedback.

Examples of expected behavior:

- assume good intent and argue from evidence
- keep criticism focused on code, behavior, risk, or product decisions
- be clear about uncertainty, tradeoffs, and reproduction steps
- respect different experience levels and communication styles

Examples of unacceptable behavior:

- harassment, abuse, threats, or personal attacks
- discriminatory language or exclusionary behavior
- publishing private information without permission
- deliberately derailing technical discussion
- spam, trolling, or repeated bad-faith engagement

## Scope

This code of conduct applies within the repository, issue tracker, pull
requests, and any project-managed discussion spaces.

## Enforcement

Project maintainers may edit, remove, or reject comments, issues, pull
requests, and other contributions that violate this code of conduct. Repeated
or severe violations may result in temporary or permanent exclusion from the
project spaces.

## Reporting

For security issues, use the process in [SECURITY.md](SECURITY.md).

For conduct issues, open a minimal issue requesting maintainer contact if no
private reporting channel is available yet. Do not include personal or sensitive
details in a public thread.
